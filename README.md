Participants (N=17, all males) with an average age of 32.8 years performed a guided visual search task in parallel with a second binaurally presented auditory task (Ries, et al., 2016). EEG data from each participant were recorded using a 64-channel BioSemi ActiveTwo system digitized at 512 Hz. Four external electrodes were used to record bipolar horizontal and vertical EOG signals, and a single external electrode was placed on each of the left and right mastoids to provide the reference signals. Fourteen participants were included in the original study, with three additional participants later added, resulting in 17 participants.

The visual search task for this experiment required participants to follow a red annulus around the screen and press a button if the annulus stopped at a prespecified target. The auditory task for this experiment was an N-back matching task in which participants listened to a string of numbers presented at approximately 2 second intervals and were required to indicate whether the current number matched a previously presented number. For the N=0, this would be the number immediately prior. For N=1 this would be the number one level before that, and so on. In the example string “1”, “1”, “2”, “1”, “3”, “2”, the second “1” should generate a match in the N=0 condition, the third “1” should generate a match in the N=1 condition, and the second “2” should generate a match in the N=2 condition. The task was composed of a baseline condition in which participants were presented with both visual and auditory stimuli but were instructed to ignore the auditory component. Next, were three dual-task conditions with N-back levels of N=0, N=1, and N=2. 

## NEMAR curation changes (2026-05-21, revised 2026-05-27)

The BIDS validator went from 17 errors + 767 warnings to 0 errors + 511 warnings. None of the raw `.set`/`.fdt` binary payloads were modified; every change is to a text sidecar.

**Task events sidecar (`task-nback_events.json`, dataset root)**
- Rewrote the sidecar to describe the columns that actually appear in the per-recording `_events.tsv` files: `onset` (with `Units` of seconds), `duration`, `sample`, and `value`. The previous version declared only a single `value` column whose `Levels` and `HED` block listed 107 trigger codes (1, 1102, 1103, 1110, and so on) that look like they came from an earlier acquisition or a sibling dataset. Every events.tsv in this rehost uses only two codes, 10 and 11, and neither one appears as a key in the old levels block, so the validator was rejecting every cell in every events.tsv as the wrong type. Replacing the sidecar with one that matches the actual columns clears the 17 events-table errors.
- Dropped the orphan `HED` dictionary along with the unmatched `Levels`. Keeping a HED annotation whose keys never appear in the data would be misleading to anyone reading the metadata. The `value` column is now described as a free-form numeric trigger code, and the two codes actually observed (10 and 11) are noted in the column's description. If a future curator recovers the correct mapping from the original lab, a HED block keyed on 10 and 11 can be added back.

**Shared recording sidecar added at the dataset root (`task-nback_eeg.json`, new)**
- Created a single root-level sidecar so the same information is not repeated in each of the 17 per-recording `_eeg.json` files; BIDS applies a root sidecar to every matching recording through inheritance. It records the electrode layout (`EEGPlacementScheme` of `10-10`, which matches the channel names Fp1, AF7, AF3, F1 through F7, FT7, FC5 through FC1, C1, and so on through O1 that every recording lists), the absence of miscellaneous and trigger channels (`MISCChannelCount` and `TriggerChannelCount` both zero, since every `channels.tsv` has 64 EEG rows after the type and units repair below and no rows of any other type), the recording hardware (`Manufacturer` of BioSemi and `ManufacturersModelName` of ActiveTwo, stated explicitly in this README's "64-channel BioSemi ActiveTwo system" line), and a `TaskDescription` paraphrased from this README. Each of these fields was previously flagged as recommended-but-missing on every recording.

**Channel tables (`sub-*/eeg/sub-*_task-nback_channels.tsv`, all 17 recordings)**
- The `type` column read `n/a` for every row; set it to `EEG` for all 64 rows in each file so the validator's channel-count rule can match the 64 EEG rows against the `EEGChannelCount` of 64 declared in each `_eeg.json`. With the type left as `n/a`, the validator saw zero EEG rows and flagged the count mismatch.
- The `units` column also read `n/a` for every row; set it to `uV` for all 64 rows, since this README documents a BioSemi ActiveTwo at 64 channels, which records microvolt-scale scalp EEG.

**Dataset description (`dataset_description.json`)**
- Updated BIDSVersion from 1.8.0 to 1.11.1 (the version the current validator checks against).
- GeneratedBy was left absent, exactly as the source published it; nothing was added there.

**Remaining warnings (511), left on purpose**
- These are all "recommended but missing" fields that need information from the study, lab, or equipment that is not in the dataset (for example: hardware filters, ground electrode, head circumference, cognitive-atlas IDs, and `GeneratedBy` on the dataset description itself). They were left blank rather than filled in with guesses.
